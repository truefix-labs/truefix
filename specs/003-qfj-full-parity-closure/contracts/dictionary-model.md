# Contract: Components, Runtime Dictionary Loading, Field Types, FIX Latest (US5, US6, US7, US9; FR-009–FR-012)

## Surface

```text
// truefix-dict::model (new directive + type; see data-model.md for full shape)
struct ComponentDef { name: String, members: Vec<ComponentMember> }

// truefix-dict (new loading API)
impl DataDictionary {
    pub fn load_from_file(path: &Path) -> Result<DataDictionary, DictLoadError>;   // NEW
    pub fn extend(&mut self, other: &DataDictionary) -> Result<(), DictMergeConflict>; // NEW
    pub fn load_fixlatest() -> Result<DataDictionary, DictLoadError>;              // NEW
}

// truefix-core::field (new constructor/accessor pairs)
impl Field {
    pub fn bytes(v: &[u8]) -> Field;                    pub fn as_bytes(&self) -> Option<&[u8]>;
    pub fn utc_date_only(v: time::Date) -> Field;       pub fn as_utc_date_only(&self) -> Option<time::Date>;
    pub fn utc_time_only(v: time::Time) -> Field;       pub fn as_utc_time_only(&self) -> Option<time::Time>;
}

// build.rs / codegen — generated the same way for every version, including fixLatest
pub fn crack_fixlatest(msg: &Message, handler: &dyn FixLatestHandler) -> Result<(), CrackError>;
```

## Behaviour

1. A `component` directive in a normalized `.fixdict` produces a `ComponentDef`; message/group member
   lists referencing `component:<Name>` expand to the component's flattened field/group list at
   dictionary-construction time (not decode time) — decode/validate code paths are unchanged.
2. `load_from_file` returns a typed error naming the path and the parse problem on failure.
   `extend` merges field/message/group/component maps; a conflicting redefinition is a typed
   `DictMergeConflict` and aborts the merge with `self` left unmodified.
3. `Field::bytes`/`as_bytes` round-trip raw bytes exactly (no UTF-8 assumption, embedded SOH preserved).
   `utc_date_only`/`utc_time_only` round-trip to the exact FIX wire format (`YYYYMMDD` /
   `HH:MM:SS[.sss...]`).
4. `load_fixlatest()` returns a dictionary converted (at build/tooling time) from FIX Orchestra XML into
   the same normalized `.fixdict` grammar every other version uses; `crack_fixlatest` and typed structs
   are generated by the same `build.rs` path as every other version — no FIX-Latest-specific runtime code.

## Acceptance (maps to spec US5/US6/US7/US9 scenarios)

- A component referenced from two messages validates identically in both (SC-005). ✔
- A runtime-loaded extension dictionary validates a message using its custom fields with no rebuild
  (SC-006). ✔
- Data/UtcDateOnly/UtcTimeOnly fields round-trip byte-identically (SC-007). ✔
- A FIX-Latest-configured session completes logon-to-heartbeat and passes its targeted AT scenarios
  (SC-008). ✔
