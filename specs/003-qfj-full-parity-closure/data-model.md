# Phase 1 Data Model: Close Remaining QuickFIX/J Parity Gaps

**Feature**: [spec.md](./spec.md) | **Research**: [research.md](./research.md)

This translates the spec's Key Entities into concrete shapes, grounded in the existing types found
during Phase 0 research. No entity here introduces a new storage system — all extend existing
`truefix-core`/`truefix-dict`/`truefix-session`/`truefix-store`/`truefix-transport` types.

## ComponentDef (extends `truefix-dict::model`)

Sibling of the existing `GroupDef` (model.rs:259-266).

```text
ComponentDef {
    name: String,
    members: Vec<ComponentMember>,
}

ComponentMember =
    Field(tag: u32)
  | Group(name: String)       // resolves to an existing GroupDef by name
  | Component(name: String)   // nested component; resolved recursively at load time
```

- **Validation rules**: a component name MUST be unique among components/messages/groups in a
  dictionary; a `Component(name)` member MUST resolve within the same dictionary (no forward reference
  to an as-yet-unmerged extension dictionary); a cycle in `Component` nesting is a typed
  `DictLoadError::ComponentCycle(name)` at load time, not a runtime/decode-time failure.
- **Lifecycle**: resolved once, at `DataDictionary` construction (parse or `extend`), into the flat
  member list `message`/`group` definitions already carry — no new runtime representation is decoded
  against; `MessageDef`/`GroupDef` are unchanged after component expansion.

## Extension Dictionary (merge semantics for `DataDictionary::extend`)

Not a new type — `extend(&mut self, other: &DataDictionary)` is a method on the existing
`DataDictionary`. Merge rule per map (`fields`, `messages`, `groups`, `components`):

- Key absent in `self` → insert from `other`.
- Key present in both, identical definition → no-op (idempotent merge).
- Key present in both, differing definition → `DictMergeConflict { key, existing, incoming }` typed
  error; the merge aborts (no partial merge state left in `self`).

## Typed Field Value (extends `truefix-core::Field` / `FieldType`)

No new enum variants — `FieldType` already declares `Data`, `UtcDateOnly`, `UtcTimeOnly` (per audit);
this feature adds the missing constructor/accessor pairs:

```text
Field::bytes(&[u8]) -> Field                     // Data (tag 95/96/212/348/352/445 pattern)
Field::as_bytes(&self) -> Option<&[u8]>

Field::utc_date_only(time::Date) -> Field        // wire: YYYYMMDD
Field::as_utc_date_only(&self) -> Option<time::Date>

Field::utc_time_only(time::Time) -> Field        // wire: HH:MM:SS[.sss...]
Field::as_utc_time_only(&self) -> Option<time::Time>
```

- **Invariant**: each accessor round-trips byte-identically with its constructor (SC-007); `as_bytes`
  performs no UTF-8 validation (Data fields may contain arbitrary bytes, including embedded SOH, per the
  length-prefixed Data-field convention already used for tag-95/96 pairs elsewhere in the dictionary).

## FIX Latest Dictionary

Not a new type — a tenth normalized `.fixdict` source file plus a `load_fixlatest()` loader function of
the same shape as the existing `load_fix44()`/etc., and a `crack_fixlatest` MessageCracker generated by
the same `build.rs` codegen path used for every other version. Its *provenance* differs (converted from
FIX Orchestra XML via a build-time `quick-xml`-based tool, §4 research) but its *runtime representation*
is identical to the other 9 versions — this is what preserves Principle IV's single-source-of-truth
invariant across all 10 versions.

## Durable Resend Source (extends `truefix-session::Session`)

```text
Session::with_store(config: SessionConfig, store: Arc<dyn MessageStore>) -> Session
```

- `build_resend(begin, end)` reads from `store.get(begin, end)` instead of the in-memory `BTreeMap`
  when a store is attached; the in-memory map remains a write-through cache (writes go to both).
- `reset()` calls `store.reset()` in addition to today's in-memory clearing, when a store is attached.
- A `Session` built via the existing (store-less) constructor is unaffected — behavior is identical to
  today.
- No change to the `MessageStore` trait itself (`truefix-store::MessageStore`, 7 async methods) — this
  entity is purely about `Session` holding and using a handle to it that it doesn't hold today.

## Extended Application Hook (extends `truefix-session::Application`)

```text
trait Application {
    // ...existing methods unchanged...
    async fn on_before_reset(&self, session_id: &SessionId) {}   // NEW, no-op default
}
```

- The "extended logon predicate" is **not** a new method — it's the existing `from_admin(&Message,
  &SessionId) -> Result<(), Reject>` hook, documented as the place arbitrary Logon-refusal logic runs
  (it already runs before the engine completes logon processing).
- **SessionStatus-carrying refusal**: the existing `Reject` outcome type gains an optional field (exact
  name TBD at task time — research.md §6 flags this as a Phase 0 follow-up, not a design fork) carrying
  a FIX `SessionStatus` (tag 573) value, propagated onto the outbound Logout/Reject the engine emits when
  `from_admin` refuses a Logon.

## Bounded Inbound Channel (extends `truefix-transport`)

```text
SessionConfig {
    // ...existing fields...
    in_chan_capacity: Option<usize>,   // NEW; None = unbounded (today's behavior, default)
}
```

Runtime shape (transport-internal, not part of the public config surface beyond the field above):

```text
admin_tx:  mpsc::UnboundedSender<Message>     // heartbeat/TestRequest/ResendRequest/Logon/Logout/...
app_tx:    mpsc::Sender<Message>              // bounded by in_chan_capacity when Some(n); unbounded channel semantics when None
```

- The socket-reading half classifies each decoded message (admin vs. application, by MsgType against
  the header/trailer definitions already known) and routes it onto the corresponding channel.
- The session processing loop drains `admin_tx`'s receiver with `select!` priority (`biased;`, admin arm
  first) ahead of `app_tx`'s receiver, so admin traffic is never blocked behind a full application queue.
- When `in_chan_capacity` is `Some(n)` and the application channel is full, the socket-reading half's
  send blocks, which stops draining the socket — backpressure propagates to the TCP receive buffer.

## AT Scenario Catalogue (extends `truefix-at`)

Not a new type — additional `Scenario`-returning functions added to `scenarios.rs`'s existing pattern,
plus `SUITE_VERSIONS` extended from `["FIX.4.2", "FIX.4.4"]` to include fix40/41/43/50/50SP1/50SP2/fixLatest
(the last gated on the FIX Latest dictionary landing), plus three new special-category suite functions
(`validate_checksum_suite()`, `timestamps_suite()`, `resynch_suite()`) alongside the existing
`server_suite()`.

## Additional SQL Backend Records (extends `truefix-store`/`truefix-log`)

No new logical entity — `SqlStore`/`SqlLog`'s existing URL-scheme dispatch (sql.rs:302-327) gains
`mssql://`/`sqlserver://` (via `tiberius`, research.md §8) and, pending the deferred driver decision,
potentially `oracle://` branches, each implementing the same internal store/log trait the
sqlx-backed branches already implement. The stored data shape (sequence numbers, message bodies,
log lines) is unchanged across backends — only the connection/query layer differs.
