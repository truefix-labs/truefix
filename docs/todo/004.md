## P0 — Critical (protocol-correctness break, data-loss risk, or DoS vector)

### BUG-23: `frame_length` integer overflow — remotely exploitable

**TF**: `crates/truefix-core/src/framing.rs:37`
```rust
let total = body_start + body_len + 7;
```
`body_len` is parsed from the wire as `usize` with no upper-bound check. A malicious sender
crafts `9=18446744073709551615`; the addition overflows. In release mode `total` wraps to a small
value, `buf.len() >= total` is true, and `frame_length` returns `Ok(Some(total))` — the caller
drains the wrong number of bytes and the stream desynchronizes. Exploitable by any unauthenticated
peer. Fix: use `checked_add`.

### BUG-24: Checksum `u32` overflow — debug panic

**TF**: `crates/truefix-core/src/codec/decode.rs:187` and `encode.rs:55`
```rust
let computed: u32 = pre.iter().map(|&b| u32::from(b)).sum::<u32>() & 0xFF;
```
`Iterator::sum::<u32>()` panics on overflow in debug builds when the message exceeds ~16.8 MB.
Violates the crate's "No path panics" invariant. Fix: use `wrapping_add` or a `u64` accumulator.

### BUG-25: Scheduled initiator never reconnects after transient connection drop

**TF**: `crates/truefix-transport/src/lib.rs`, `run_scheduled_initiator` (~L960-995).
`connect_initiator_with` → `spawn_session_io` spawns `run_connection` in a separate tokio task
and returns the `SessionHandle` immediately. The `JoinHandle` is never awaited, so when the
connection drops `current` remains `Some(handle)` forever. The reconnect condition
`current.is_none()` never becomes true. Directly contradicts the code's own comment ("keep
retrying a connection while in session and disconnected, e.g. after a transient network drop").
Distinct from 003.md BUG-09 (spurious store reset on restart — same function, different bug).

### BUG-26: SQL Store migration-order bug — breaks upgrading pre-existing databases

**TF**: `crates/truefix-store/src/sql.rs`, `SqlStore::ensure_schema` (~L230-285).
`INSERT ... creation_time ...` for the sessions table runs **before**
`add_creation_time_column_if_missing()`. For a sessions table created before the `creation_time`
column existed, `CREATE TABLE IF NOT EXISTS` is a no-op and the INSERT references a non-existent
column. `ON CONFLICT`/`IGNORE` suppresses constraint violations, not schema errors. Affects all
three SQL backends (SQLite/Postgres/MySQL). Fix: call `add_creation_time_column_if_missing()`
before the INSERT.

### BUG-27: `Engine::shutdown()` cannot stop regular initiators + no `Drop` impl

**TF**: `crates/truefix/src/lib.rs`, `Engine::shutdown` (~L420-426) and `Engine` struct.
`shutdown()` aborts `self.acceptors` and stops `self.failover_initiators`, but never touches
`self.initiators` (`Vec<SessionHandle>`). `SessionHandle` exposes only `logout()` (async) and
`join(self)` — its inner `JoinHandle` is private with no `abort`. So even an explicit
`shutdown()` cannot terminate these tasks. Additionally, `Engine` has no `Drop` impl — if a
caller drops `Engine` without calling `shutdown()`, every spawned tokio task is detached
(acceptors keep accepting, initiators keep running, failover loops keep reconnecting forever).
Distinct from 003.md BUG-21 (which notes the doc-comment asymmetry; this is the stronger finding
that there is no available API to clean up at all).

### BUG-28: ResetSeqNumFlag handshake completely broken

**TF**: `crates/truefix-session/src/admin.rs` `logon()` + `crates/truefix-session/src/state.rs` `on_logon()`.

Two compounding defects:
1. **Acceptor does not echo `ResetSeqNumFlag=Y`**: `admin::logon()` sets the flag based on
   `config.reset_on_logon` (a static config), not on whether the incoming Logon had it. QFJ's
   `generateLogon(otherLogon, ...)` sets it based on `state.isResetReceived()`.
2. **Initiator does not verify the echo, nor infer from seq=1 (QFJ-383)**: `on_logon()` has no
   `reset_sent`/`reset_received` tracking. QFJ disconnects with "Expected Logon response to have
   reset sequence numbers" if the acceptor didn't acknowledge, and infers the flag from seq=1
   when absent.

Combined: the entire ResetSeqNumFlag mechanism is non-functional. Related to 003.md B1 (which
covers the `full = !self.logon_sent` partial-reset issue), but the echo/verification/inference
defects are distinct.

### BUG-29: ResendRequest with too-high sequence number deferred — potential deadlock (QFJ-673)

**TF**: `crates/truefix-session/src/state.rs`, `on_received()` `Ordering::Greater` branch.
A too-high ResendRequest is queued and `build_resend()` is NOT called. The ResendRequest is only
processed later from `drain_queue()` when the gap is filled. QFJ's `nextResendRequest()` calls
`verify(rr, false, validateSequenceNumbers)` — deliberately skipping the too-high check (QFJ-673)
and processing the resend immediately. Deadlock scenario: we expect seq 3; counterparty sends
ResendRequest(seq=5); we queue it and send our own ResendRequest for 3–4; counterparty waits for
our response to their ResendRequest(5); we wait for them to fill our gap(3–4) — deadlock.

### BUG-30: `SeqFile::write` uses truncate-then-write — crash loses both sequence numbers

**TF**: `crates/truefix-store/src/file.rs`, `SeqFile::write`.
Opens the seqnums file with `.truncate(true)` — zeros the file before writing. This runs on
every `set_next_sender_seq`/`set_next_target_seq` call (i.e., every message). A crash between
truncate and write leaves the file empty. QFJ uses `seek(0)` + `writeUTF()` without truncation,
and keeps sender/target in separate files. Distinct from 003.md B17 (which is about
`FileStore::reset()` non-atomicity; this affects the regular per-message write path).

### BUG-31: Corrupted seqnums file silently defaults to (1, 1)

**TF**: `crates/truefix-store/src/file.rs`, `load_seqnums`.
`unwrap_or(1)` on parse failure silently defaults to 1. Combined with BUG-30, a crash guarantees
silent corruption: the session restarts from (1, 1) with no error logged. QFJ's
`initializeSequenceNumbers()` throws `IOException` on corrupted data, preventing session startup.

### BUG-32: No duplicate/competing connection protection on acceptor

**TF**: `crates/truefix-transport/src/lib.rs`, `route_and_run` / `Acceptor::serve`.
QFJ's `AcceptorIoHandler` checks `qfSession.getResponder() != null` when a Logon arrives — if
the session is already bound to an active connection, it closes the new one. TrueFix has no such
check. A second connection presenting the same SessionID creates a competing session, leading to
sequence-number corruption and duplicate message processing.

### BUG-33: `reset_on_disconnect` not honored when TCP connection drops

**TF**: `crates/truefix-transport/src/lib.rs`, `run_connection`.
When the TCP connection drops, `run_connection` exits and calls `write_half.shutdown()` +
`on_logout`, but **never dispatches any event to the `Session` state machine**. The `Session` is
simply dropped. `enter_disconnected()` (which checks `reset_on_disconnect || reset_on_logout`)
is only called from within the state machine on Logout-received/logon-timeout/heartbeat-timeout.
The `Event` enum has no `Disconnected` variant. For the reconnecting initiator, the store
retains stale sequence numbers; `ResetOnDisconnect` is silently ignored.

### BUG-34: `from_app`/`from_admin` called before session-level validation

**TF**: `crates/truefix-transport/src/lib.rs`, `process_inbound` (~L970-990).
TrueFix calls `from_admin`/`from_app` **before** dispatching to the session state machine. QFJ
calls `fromCallback()` inside `verify()` — after all session-level checks (sequence, identity,
latency, PossDup) pass. Consequence: application callbacks receive messages that the session
layer hasn't validated (gap fill not yet processed, dictionary validation not yet run, identity
not yet confirmed). If the message is later rejected or the session is torn down, the application
has already processed it.

## P1 — High/Medium (real defect, narrower blast radius)

### BUG-35: `resend_requested` not reset on reconnect — suppresses new ResendRequest

**TF**: `crates/truefix-session/src/state.rs`, `on_connected()`.
`on_connected` does not reset `resend_requested` or `queue`. 003.md B4 covers
`resend_target`/`resend_chunk_end`/timers, but the `resend_requested` flag and `queue` are also
stale. On reconnect, `on_logon`'s `Ordering::Greater` arm is guarded by
`if !self.resend_requested` — a new gap will not trigger a ResendRequest. The session silently
loses the gap.

### BUG-36: `on_tick` heartbeat-timeout arithmetic can overflow `u32`

**TF**: `crates/truefix-session/src/state.rs`, `on_tick()` `LoggedOn` arm.
`hb * self.config.heartbeat_timeout_multiplier.max(1) + 2` is plain `u32` arithmetic. Large values
overflow → panic in debug, silent wrap in release. Additionally,
`heartbeat_timeout_multiplier` is parsed as `u32` (losing fractional precision — QFJ uses
`double`, default 1.4). A `.cfg` with `HeartBeatTimeoutMultiplier=1.4` truncates to 1, yielding a
2.25× shorter timeout than QFJ. Fix: use `saturating_mul`/`saturating_add` and change the field
type to `f64`.

### BUG-37: `decode_with_groups` never sets `fields_out_of_order`

**TF**: `crates/truefix-core/src/codec/decode.rs:76-96`.
`decode()` performs section-ordering validation (sets `fields_out_of_order = true`); the
group-aware `decode_with_groups` does none of this. A grossly out-of-order message decoded via
this path reports `fields_out_of_order() == false` unconditionally. 003.md GAP-26 notes the
function is dormant, but doesn't track that even if called, the ordering check is missing.

### BUG-38: Data field tag not verified against the expected data tag

**TF**: `crates/truefix-core/src/codec/decode.rs:251`.
`data_field_for_length(tag)` returns the expected data tag (e.g. 95→96), but only `.is_some()`
is checked — the return value is discarded. If a non-data field follows a length field, the
parser silently consumes `len` bytes from the wrong field, corrupting all subsequent parsing.
003.md B22 is about missing length/data pairs in the whitelist; this is about the verification
logic itself being broken for existing pairs.

### BUG-39: Plain reconnecting initiator skips socket options

**TF**: `crates/truefix-transport/src/lib.rs`, `connect_initiator_reconnecting_multi` (~L885-915).
On a successful connect it calls `run_connection(stream, ...)` without calling
`services.socket_options.apply(&stream)`. The TLS variant correctly applies them. The plain
reconnecting initiator never sets `tcp_no_delay`, `keep_alive`, buffer sizes, etc.

### BUG-40: `logon_sent` not reset on AwaitingLogon timeout

**TF**: `crates/truefix-session/src/state.rs`, `on_tick()` → `enter_disconnected`.
When logon timeout fires and `reset_on_disconnect`/`reset_on_logout` are not configured,
`enter_disconnected` does not reset `logon_sent`. On reconnect, `seed_sequences` is a no-op
(guarded by `if !self.logon_sent`), so the transport cannot re-seed sequence numbers from the
durable store.

### BUG-41: MSSQL `save_and_advance_sender` does not rollback on COMMIT failure

**TF**: `crates/truefix-store/src/mssql.rs`, `save_and_advance_sender` (~L290-330).
If `COMMIT TRANSACTION` returns an error, the code returns `Err(e)` but never issues `ROLLBACK`,
leaving an open transaction on the mutex-held connection. The error branch (statement failure)
correctly rolls back; only the COMMIT-failure case is mishandled.

### BUG-42: No `validLogonState` check — messages processed before logon completes

**TF**: `crates/truefix-session/src/state.rs`, `on_received()`.
`on_received()` has no state check. A non-Logon message arriving in `AwaitingLogon` state is
processed normally — if its seq is too high, TrueFix queues it and sends a ResendRequest
(protocol violation: no ResendRequests before logon). QFJ's `validLogonState()` rejects non-
Logon/non-Logout/non-SequenceReset/non-Reject messages when not logged on, causing a disconnect.

### BUG-43: `NextExpectedMsgSeqNum(789)` too-high not rejected

**TF**: `crates/truefix-session/src/state.rs`, `on_logon()` (~L900).
TrueFix only handles the too-low case (resends the gap). If 789 > `next_out_seq` (counterparty
expects messages we never sent), TrueFix silently ignores it, leaving the session desynced. QFJ
disconnects with "Tag 789 is higher than expected."

### BUG-44: Logon without `MsgSeqNum` (tag 34) is silently accepted

**TF**: `crates/truefix-session/src/state.rs`, `on_logon()` (~L835).
`logon_seq` is `None` when tag 34 is absent; `disposition` is `None`; the `Some(Ordering::Equal)`
guard doesn't fire, so `next_in_seq` is not advanced. But the state still transitions to
`LoggedOn` and a Logon response is sent. The session is now desynced. QFJ checks `MsgSeqNum`
presence at the top of `next()` for all messages.

### BUG-45: `load_index` reads entire body file into memory — OOM on open

**TF**: `crates/truefix-store/src/file.rs`, `load_index` (called by `BodyLog::open`).
`fs::read(&path)` reads the entire body file (all message bytes ever stored) into a `Vec<u8>`
to build the offset index. For a long-running session with 1M messages at ~500 bytes each, this
allocates ~500 MB just to open. QFJ's `FileStore` reads only a separate compact header/index
file (16 bytes per message) and never touches the body file at open time.

### BUG-46: `frame_length` does not verify CheckSum field at the computed position

**TF**: `crates/truefix-core/src/framing.rs`, `frame_length()`.
Computes `total = body_start + body_len + 7` and returns `Ok(Some(total))` whenever
`buf.len() >= total` — without checking that bytes at `body_start + body_len` start with `10=`.
A wrong BodyLength makes `total` point at the wrong position; the transport drains the wrong
number of bytes and desynchronizes. QFJ's `FIXMessageDecoder` verifies `10=???<SOH>` at the
expected position and rescans on mismatch.

### BUG-47: `as_int()`/`as_decimal()` trim whitespace QFJ rejects

**TF**: `crates/truefix-core/src/field.rs`, `as_int`/`as_decimal`.
Uses `s.trim().parse()`, accepting leading/trailing whitespace (including Unicode whitespace)
and `+`-prefixed integers. QFJ's `IntConverter`/`DecimalConverter` do not trim and reject these.

### BUG-48: Timestamp parsing accepts non-standard fractional digit counts

**TF**: `crates/truefix-core/src/field.rs`, `parse_utc_timestamp`/`parse_utc_time_only`.
Accepts any number of fractional digits ≥ 1. QFJ accepts only 0/3/6/9/12 digits (timestamp) or
0/3/6/9/12 (time-only). `20240101-12:00:00.1` (1 digit) and 13+ digits are accepted by TrueFix
but rejected by QFJ.

### BUG-49: `tokenize` silently ignores invalid data-length field values

**TF**: `crates/truefix-core/src/codec/decode.rs`, `tokenize`.
When a length field (e.g. tag 95) has a non-numeric value, `parse_usize(&value)` returns `None`,
`pending_data_len` is never set, and the following data field is parsed as normal text with no
error. QFJ throws `FieldConvertError` → `InvalidMessage`.

### BUG-50: `ReconnectHandle::stop()` cannot stop a currently-connected initiator

**TF**: `crates/truefix-transport/src/lib.rs`, `ReconnectHandle::stop`.
`stop()` only sets an `AtomicBool`. The reconnect loop checks it at the top of the `while` loop
and after `run_connection` returns. But if a connection is currently active, the loop is blocked
inside `run_connection`. `ReconnectHandle` doesn't hold the `control_tx` sender. There is no
`logout()` method either. The initiator keeps running indefinitely until the connection drops.

### BUG-51: Multi-endpoint rotation not reset after successful connect

**TF**: `crates/truefix-transport/src/lib.rs`, `connect_initiator_reconnecting_multi`.
`next_endpoint` is incremented before each attempt and never reset to 0 after a successful
connect. After a connection to endpoint N drops, the next reconnect tries N+1 (wrapping) instead
of preferring the primary. QFJ resets `nextSocketAddressIndex = 0` on successful connect.

### BUG-52: Acceptor does not validate that the first message is a Logon

**TF**: `crates/truefix-transport/src/lib.rs`, `route_and_run`.
`route_and_run` reads the first complete message, decodes it, and uses its comp-ID fields for
routing — but never checks `MsgType == A`. A non-Logon first message is routed, potentially
creating a session from a dynamic template with incorrect comp IDs. QFJ refuses immediately.

### BUG-53: Unbounded admin channel — no backpressure safety valve

**TF**: `crates/truefix-transport/src/lib.rs`, `read_loop`.
Uses `mpsc::unbounded_channel()` for all administrative traffic. If the processing loop is
stalled (e.g. slow `from_app`), the admin channel grows without bound — OOM. QFJ uses bounded
queues (capacity 10000) with backpressure.

### BUG-54: Group required fields not recursively validated

**TF**: `crates/truefix-dict/src/validate.rs`, `validate()`.
QFJ's `checkHasRequired` recursively enters each group entry and checks the group's own
`requiredFields`. TrueFix only checks the message body's `mdef.required` — it never recursively
validates required fields within group entries. A group entry missing a required delimiter is
not caught (if the count declares correctly).

### BUG-55: Group field types not validated for group-decoded messages

**TF**: `crates/truefix-dict/src/validate.rs`, `validate_groups()`.
QFJ's `iterate()` recurses into `map.getGroups()` and validates each group entry's fields via
the group's own `DataDictionary`. TrueFix's `validate_groups` only works for flat-decoded
messages (where `body.fields()` yields group members). If decoded via `decode_with_groups`,
group members are in `Member::Group` and `body.fields()` skips them — these fields are never
type/enum-checked.

### BUG-56: Outbound messages do not include SubID/LocationID

**TF**: `crates/truefix-session/src/admin.rs` `base()` + `crates/truefix-session/src/state.rs` `send_app()`.
GAP-47 (feature 005) added `sender_sub_id`/`sender_location_id`/`target_sub_id`/
`target_location_id` to `SessionConfig`, but `admin::base()` and `send_app()` never use them.
All outbound messages are missing SubID(50/142) and LocationID(57/143). QFJ's `initializeHeader`
sets all six identity fields. Configured sessions emit messages the counterparty cannot route.

### BUG-57: PossDup anti-replay check only executes on `Ordering::Less`

**TF**: `crates/truefix-session/src/state.rs`, `on_received()`.
`poss_dup_anti_replay_violation` is only called in the `Ordering::Less` branch. QFJ's
`validatePossDup` runs for all `PossDupFlag=Y` messages (including expected-seq and too-high).
A `PossDupFlag=Y` message with `OrigSendingTime > SendingTime` and seq == expected is not
rejected by TrueFix.

### BUG-58: Reject(35=3) missing `RefMsgType(372)`

**TF**: `crates/truefix-session/src/admin.rs`, `reject_with_reason()` and `reject()`.
FIX 4.2+ requires Reject to include `RefMsgType(372)`. QFJ sets it when
`beginString >= FIX.4.2`. TrueFix never sets it on any Reject.

### BUG-59: `SessionRejectReason(373)` not version-filtered

**TF**: `crates/truefix-session/src/admin.rs`, `reject_with_reason()`.
QFJ filters by version: FIX ≤ 4.1 doesn't set the field; FIX 4.2 only codes ≤ 11; FIX 4.3 only
≤ 15; FIX 4.4 only ≤ 16 or `OTHER`. TrueFix unconditionally sets it for all versions — a
protocol violation for FIX 4.0/4.1.

### BUG-60: Logout response does not echo received `Text(58)`

**TF**: `crates/truefix-session/src/state.rs`, `on_logout_msg()`.
When receiving a Logout, TrueFix's response has `text = None`. QFJ's `generateLogout(otherLogout)`
copies `LastMsgSeqNumProcessed` from the received Logout and the transport echoes text.

### BUG-61: Acceptor-group `Services`/store built only from the first member

**TF**: `crates/truefix/src/lib.rs`, acceptor-group block (~L255-330).
The shared `Services` (store, log, socket_options, validator) is built exclusively from
`primary = members.first()`. If two sessions sharing a port configure different
`FileStorePath`/`JdbcURL`, the second session's messages/sequence numbers are written to the
first session's store — silent state corruption. 003.md BUG-16 covers
`ContinueInitializationOnError` taking the first member's flag; this is the store/log/validator
silently using the wrong session's storage.

### BUG-62: `CHAR` field validation not version-sensitive

**TF**: `crates/truefix-dict/src/model.rs`, `FieldType::value_ok`, `Self::Char` arm.
QFJ skips `CharConverter` validation for FIX ≤ 4.1 (treats CHAR as String). TrueFix
unconditionally calls `field.as_char()` for all versions — FIX 4.0/4.1 CHAR fields with
multi-character values are rejected by TrueFix but accepted by QFJ.

## P2 — Low (minor, low-probability, or tooling/hygiene)

### BUG-63: `precision_key` is case-sensitive

**TF**: `crates/truefix-config/src/builder.rs`, `precision_key` (~L355-372).
Requires exact uppercase (`SECONDS`/`MILLIS`/`MICROS`/`NANOS`). Every other parser in the same
file is case-insensitive. `TimeStampPrecision=seconds` fails with `InvalidValue`.

### BUG-64: `bool_key` silently treats any non-"Y" value as `false`

**TF**: `crates/truefix-config/src/builder.rs`, `bool_key` (~L245-249).
`SocketUseSSL=true` silently disables TLS. No error for invalid values. May mirror QFJ's strict
Y/N grammar, but the silent acceptance of arbitrary strings as `false` is a robustness concern.

### BUG-65: `parse_utc_offset` integer overflow

**TF**: `crates/truefix-config/src/builder.rs`, `parse_utc_offset` (~L810-820).
`hours * 3600` can exceed `i32::MAX` for large values — debug panic.

### BUG-66: `prepare_resend` ignores `timestamp_precision`

**TF**: `crates/truefix-session/src/admin.rs`, `prepare_resend`.
The refreshed `SendingTime` calls `now_utc_timestamp()` (hardcoded `Milliseconds`), not
`now_utc_timestamp_prec(self.config.timestamp_precision)`. Resent messages always use
millisecond precision regardless of session configuration.

### BUG-67: Acceptor truncates peer's `HeartBtInt` via `as u32`

**TF**: `crates/truefix-session/src/state.rs`, `on_logon()`, `Role::Acceptor` arm.
`self.config.heartbeat_interval = hbi as u32` where `hbi` is `i64`. A peer-supplied value >
`u32::MAX` silently truncates to garbage. Should clamp.

### BUG-68: `MsgSeqNum too low` path skips `reset_on_error()`

**TF**: `crates/truefix-session/src/state.rs`, `on_received()`, `Ordering::Less` non-PossDup branch.
Latency-failure and identity-failure paths both call `reset_on_error()` after disconnecting. The
hard sequence-violation path does not — inconsistent if `ResetOnError` is configured.

### BUG-69: `on_tick` `AwaitingLogout` does not send heartbeats

**TF**: `crates/truefix-session/src/state.rs`, `on_tick()`.
`AwaitingLogout` (not yet timed out) falls through to `_ => {}` — no heartbeats. If
`logout_timeout` is long, the peer may disconnect us for heartbeat failure before our timeout.

### BUG-70: MSSQL URL parser breaks on passwords containing `@`

**TF**: `crates/truefix-store/src/mssql.rs` + `crates/truefix-log/src/mssql.rs`, `parse_url`.
`split_once('@')` splits at the first `@`. A password containing `@` is mis-split. 003.md GAP-55
is about `splice_credentials` not percent-encoding — this is a different function (`parse_url`)
with a different root cause.

### BUG-71: `tcp_connect` blocking-connect thread/socket leak on timeout

**TF**: `crates/truefix-transport/src/lib.rs`, `tcp_connect` (~L175-200).
When `local_bind_addr` is set, `spawn_blocking` runs a blocking `connect()`. On timeout, the
future is dropped but the `spawn_blocking` thread keeps running until the OS TCP timeout.

### BUG-72: codegen `type_mapping` omits `TIME`/`PRICEOFFSET` and other US9 types

**TF**: `crates/truefix-dict/src/codegen.rs`, `type_mapping`.
Only maps `INT`/`FLOAT`/`CHAR`/`BOOLEAN`/`UTCTIMESTAMP` explicitly; everything else falls to
`&str`. `TIME` (same wire format as `UtcTimestamp`) and `PRICEOFFSET` (decimal) get string
accessors instead of typed ones.

### BUG-73: codegen enum-variant labels that are Rust keywords are not sanitized

**TF**: `crates/truefix-dict/src/codegen.rs`, `sanitize_variant`.
Only prefixes leading digits. A label equal to a Rust keyword (`Type`, `Self`, `Match`, etc.)
produces uncompilable generated code.

### BUG-74: codegen vs runtime track divergence on `open` enum modifier

**TF**: `crates/truefix-dict/src/codegen.rs` vs `crates/truefix-dict/src/parser.rs`.
Runtime parser recognizes and strips a bare `open` token after the field type. Codegen's
`parse_dict` does not — it would ingest `open` as an enum value. Dual-track divergence.

### BUG-75: `on_logon` in `Disconnected` state still processes sequence logic

**TF**: `crates/truefix-session/src/state.rs`, `on_logon()`.
If a Logon arrives when `state == Disconnected` (transport race), the role/state guards fail
(`_ => {}`), so no logon response is sent. But the `disposition` / `NextExpectedMsgSeqNum`
logic still runs, potentially advancing `next_in_seq` or issuing a spurious ResendRequest on a
session that isn't actually connected.

### BUG-76: Acceptor-group TLS can come from a different member than the store/Services

**TF**: `crates/truefix/src/lib.rs` (~L300).
`Services` is sourced from `members.first()`; TLS is sourced from `members.iter().find_map(...)`
— the first member that has TLS. When the primary has no TLS but a later member does, the listener
ends up with TLS material from one session and the store/log/validator from a different one.

### BUG-77: `Monitor` never wired by `Engine::start`

**TF**: `crates/truefix/src/lib.rs` (~L285,345).
`Services` has a `monitor: Option<Monitor>` field that `Engine::start` always leaves as `None`.
The transport's operational API (`force_logout`/`reset`/`send_app`/`status`/`is_connected`) is
unreachable for any session started through the engine facade.

### BUG-78: Timestamp parsing rejects leap seconds (sec=60) that QFJ accepts

**TF**: `crates/truefix-core/src/field.rs`, `parse_utc_timestamp`.
`time::Time::from_hms_nano` rejects sec=60. QFJ explicitly maps `s == 60` → `s=59, ns=999999999`.

### BUG-79: `frame_length` does not validate BeginString format

**TF**: `crates/truefix-core/src/framing.rs`, `frame_length()`.
Only checks `buf.get(0..2) == Some(b"8=")`. QFJ requires `8=FIX.X.X` or `8=FIXT.X.X` pattern.
Non-FIX data is framed by TrueFix.

### BUG-80: `decode` accepts messages with no `MsgType` / `BodyLength=0`

**TF**: `crates/truefix-core/src/codec/decode.rs`, `tokenize_validated`.
Only requires field 1=BeginString, field 2=BodyLength. Does not require field 3=MsgType.
`8=FIX.4.4\x019=0\x0110=086\x01` (no MsgType) is accepted.

### BUG-81: `CachedFileStore::open_with_options` materializes all message bodies on open

**TF**: `crates/truefix-store/src/file.rs`, `CachedFileStore::open_with_options`.
`body.all()` reads every stored message body into a `Vec` before cache insertion. Peak memory
is ALL message bodies, not `max_cached_msgs`. QFJ's `CachedFileStore` never reads bodies on open.

### BUG-82: Stale session status in `Monitor` after connection drop

**TF**: `crates/truefix-transport/src/lib.rs`, `run_connection` shutdown path.
`mark_disconnected` sets `connected: false` but does not update `status`. `monitor.status()`
returns `LoggedOn` while `is_connected()` returns `false` — inconsistent snapshot. Consequence
of BUG-33 (session state machine never notified of disconnection).

### BUG-83: GapFill `SequenceReset` missing `OrigSendingTime(122)`

**TF**: `crates/truefix-session/src/admin.rs`, `sequence_reset()`.
QFJ's `generateSequenceReset` sets `OrigSendingTime = SendingTime`. TrueFix does not. Strict
peers may require `OrigSendingTime` on all `PossDupFlag=Y` messages.

### BUG-84: FIX < 4.2 not forced to seconds-precision timestamps

**TF**: `crates/truefix-session/src/admin.rs` `base()` + `state.rs` `send_app()`.
QFJ forces `UtcTimestampPrecision.SECONDS` for `beginString < FIX.4.2`. TrueFix always uses
`config.timestamp_precision` (default `Milliseconds`) — FIX 4.0/4.1 sessions emit
`YYYYMMDD-HH:MM:SS.sss`, a protocol violation for those versions.

### BUG-85: Acceptor also gets Logon timeout

**TF**: `crates/truefix-session/src/state.rs`, `on_tick()`.
`AwaitingLogon` timeout applies to both roles. QFJ's `next()` only times out when
`state.isLogonAlreadySent()` is true (initiator only). Acceptors wait indefinitely for a Logon.

#### BUG-36: `on_tick` heartbeat-timeout arithmetic can overflow `u32` — ⚠️ Partially confirmed

- **Overflow risk**: Confirmed. `hb * self.config.heartbeat_timeout_multiplier.max(1) + 2` is plain `u32` arithmetic; large values overflow → panic in debug, silent wrap in release.
- **`heartbeat_timeout_multiplier` is `u32`**: Confirmed. `config.rs` declares `pub heartbeat_timeout_multiplier: u32`. `builder.rs` parses it via `u32_key`. A `.cfg` value of `1.4` truncates to `1`.
- **Directional error in original description**: The text states "yielding a 2.25× shorter timeout than QFJ." This is **wrong in direction**. TrueFix's default `heartbeat_timeout_multiplier` is `2` (giving `2×HB+2`), while QFJ's default is `1.4` (giving `1.4×HB+2`). TrueFix's timeout is **longer**, not shorter. If a user sets `HeartBeatTimeoutMultiplier=1.4`, it truncates to `1`, yielding `1×HB+2` — which *is* shorter than QFJ's `1.4×HB+2`, but the original text's "2.25× shorter" figure is incorrect regardless.

#### BUG-44: Logon without `MsgSeqNum` (tag 34) is silently accepted — ⚠️ Partially confirmed

- **Core issue confirmed**: When tag 34 is absent, `logon_seq_early` is `None`, `disposition` is `None`, and the `Some(Ordering::Equal)` guard does not fire. `next_in_seq` is not advanced. The session still transitions to `LoggedOn` and a Logon response is sent (acceptor arm). The session is desynced.
- **Description nuance**: The text says "`Some(Ordering::Equal)` guard doesn't fire" — technically correct but incomplete. When `disposition` is `None`, *none* of the three match arms (`Some(Equal)`, `Some(Greater) if !resend_requested`, `_`) match except the catch-all `_ => {}`. The effect is the same but the mechanism is broader than described.

#### BUG-56: Outbound messages do not include SubID/LocationID — ❌ Not confirmed (partially)

- **`admin::base()` DOES set SubID/LocationID**: The original description claims "`admin::base()` and `send_app()` never use them." This is **factually incorrect for `base()`**. The function at `admin.rs:14-38` explicitly sets tags 50/142/57/143 when configured:
  ```rust
  if let Some(v) = &config.sender_sub_id { m.header.set(Field::string(50, v)); }
  if let Some(v) = &config.sender_location_id { m.header.set(Field::string(142, v)); }

  ```
  All admin messages (Logon, Logout, Heartbeat, Reject, etc.) go through `base()` and correctly carry these fields.
- **`Session::send_app()` does NOT set them**: `state.rs` `send_app()` stamps the header manually (tags 8/49/56/34/52) without calling `base()` and without setting 50/142/57/143. So application-originated messages are indeed missing SubID/LocationID.

#### BUG-65: `parse_utc_offset` integer overflow — ⚠️ Partially confirmed

- **Overflow path exists**: `sign * (hours * 3600 + minutes * 60)` uses `i32` arithmetic. `hours` is parsed as `i32`. The overflow threshold is `hours > ~596523`.
- **Practical risk is negligible**: A valid UTC offset's `hours` field is 0–14. Even a malicious `.cfg` value like `+99999:00` would parse `hours = 99999`, giving `99999 * 3600 = 359996400` which fits in `i32` (max ~2.1 billion). You'd need `hours > 596523` to overflow — an extreme, non-realistic input. The debug-panic risk is theoretical.

#### BUG-73: codegen enum-variant labels that are Rust keywords are not sanitized — ❌ Not confirmed (practically)

- **Code analysis is correct**: `sanitize_variant()` only prefixes leading digits. Enum variant labels do NOT pass through `snake_case()` (which has keyword sanitization). So a label like `Type` or `Self` would indeed produce invalid Rust.
- **Doesn't trigger in practice**: The code's own comments note "real QFJ-schema labels are always `[A-Za-z0-9_]+`" and use SCREAMING_SNAKE_CASE (e.g. `PER_UNIT`, `BUY`, `SELL`). No real QFJ enum label is a Rust keyword. The risk is theoretical for bundled dictionaries; it could only trigger on a custom dictionary source that names an enum value `Type` or `Self`.

## P0 — Critical (QFJ/QFGo comparison)

### BUG-86: Session state machine has no schedule awareness — mid-connection schedule violations go unenforced

**TF**: `crates/truefix-session/src/state.rs`, `crates/truefix-transport/src/lib.rs`.
**QFJ**: `Session.java` `next()` L1961-1976. **QFGo**: `session_state.go` `CheckSessionTime()`.

QFJ's `Session.next()` checks `isSessionTime()` every second and resets/disconnects if outside the
schedule window — for both initiator and acceptor. `resetIfSessionNotCurrent` (L3139) compares
`store.getCreationTimeCalendar()` against `sessionSchedule.isSameSession()`. QFGo calls
`CheckSessionTime` on **every incoming message** in addition to the per-second ticker.

TrueFix's schedule logic lives only in `transport/lib.rs` `run_scheduled_initiator` — the acceptor
has zero schedule awareness. The session state machine (`state.rs`) has no concept of schedule at all.

**Impact**: An acceptor that should only be active during trading hours will accept Logons at 3am
and keep the session alive. A daily-reset schedule is silently ignored on the acceptor side. ✅ **Confirmed**

### BUG-87: Acceptor does not reject Logon outside session schedule

**TF**: `crates/truefix-session/src/state.rs` `on_logon()`.
**QFJ**: `Session.java` `nextLogon()` L2182-2183. **QFGo**: `session.go` `onAdmin()`.

QFJ: `if (!isSessionTime()) { throw new RejectLogon("Logon attempt not within session time"); }`
QFGo: `if !s.IsSessionTime() { return errors.New("Connection outside of session time") }`

TrueFix's `on_logon()` has no schedule check. ✅ **Confirmed**

### BUG-88: `forceResendWhenCorruptedStore` semantics diverge — admin messages not resent during gap-fill

**TF**: `crates/truefix-session/src/state.rs` `build_resend()`.
**QFJ**: `Session.java` `resendMessages()` L2386-2392.

gap-filled when the flag is true; (2) on `IOException` reading from store, heartbeats are sent
instead of throwing. TrueFix only uses this flag at store-open time. ✅ **Confirmed**

### BUG-97: ResendRequest `EndSeqNo=999999` not treated as "infinity" for FIX ≤ 4.1

## 生产发布建议

### 第一优先级 — 必须在上线前修复 (15 项，原 18 项，二次核实后 BUG-23/98/101 移出)

这些 bug 会导致以下任一后果：
- **数据丢失**：BUG-30, 31, 26, 41
- **协议破坏**：BUG-28, 29, 34, 89
- **资源泄漏/无法关闭**：BUG-25, 27, 32, 33
- **安全/DoS**：BUG-100
- **调度违规**：BUG-86, 87

大部分是一行到几行的修复。最复杂的是 BUG-28（ResetSeqNumFlag 握手）和 BUG-34（callback 排序），需要重构 `on_logon` 和 `handle_inbound` 的调用顺序。

### 第二优先级 — 建议在第一个生产版本后立即修复 (31 项)

这些是真实 bug 但有 workaround 或需特殊条件触发。可在上线后第一个迭代中修复。

### 第三优先级 — 随常规开发进度修复 (33 项)

低概率、工具链问题、或架构差异。不影响正确性。

### 关于 BUG-24 的特别说明

Checksum `u32` 溢出在 **release** 构建中是无害的——`Iterator::sum::<u32>()` 在 release 模式下使用 wrapping add，而最终的 `& 0xFF` 使得溢出对结果无影响（因为 `(a + b) mod 256` 与 `((a mod 256) + (b mod 256)) mod 256` 在数学上等价，无论中间是否溢出）。**仅 debug 构建会 panic**。

如果生产环境使用 release 构建编译，BUG-24 不阻塞上线。但建议修复以满足 "No path panics" 的 crate 级别 invariant。

### 关于 BUG-111 的特别说明

TrueFix 的 "每连接创建新 Session" 架构是**有意的设计选择**，不是 bug。它简化了状态管理（无需锁、无需跨线程同步），代价是失去了 QFJ 的 `Responder` swap 机制带来的能力（重复连接检测、`validLogonState` 等）。

这些缺失的能力已被单独追踪为 BUG-32（重复连接）、BUG-42（`validLogonState`）、BUG-33（`onLogout` on TCP drop）。BUG-111 本身不需要单独修复——它是其他 bug 的架构根因，不是独立的可操作项。
